## Create Endpoint

<CodeGroup>
```bash cURL - Webhook
curl -X POST https://api.inbound.new/api/e2/endpoints \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Support Webhook",
    "type": "webhook",
    "config": {
      "url": "https://api.example.com/webhooks/email",
      "timeout": 30,
      "retryAttempts": 3,
      "headers": {
        "X-Custom-Header": "value"
      }
    },
    "description": "Webhook for support emails"
  }'
```

```bash cURL - Email Forward
curl -X POST https://api.inbound.new/api/e2/endpoints \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Forward to Gmail",
    "type": "email",
    "config": {
      "forwardTo": "team@gmail.com",
      "preserveHeaders": true
    }
  }'
```

```bash cURL - Email Group
curl -X POST https://api.inbound.new/api/e2/endpoints \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Team Distribution List",
    "type": "email_group",
    "config": {
      "emails": [
        "alice@team.com",
        "bob@team.com",
        "carol@team.com"
      ],
      "preserveHeaders": true
    }
  }'
```

```typescript TypeScript SDK
import { Inbound } from '@inbound/sdk'

const inbound = new Inbound({ apiKey: 'YOUR_API_KEY' })

// Webhook endpoint
const webhook = await inbound.endpoints.create({
  name: 'Support Webhook',
  type: 'webhook',
  config: {
    url: 'https://api.example.com/webhooks/email',
    timeout: 30,
    retryAttempts: 3,
    headers: {
      'X-Custom-Header': 'value'
    }
  },
  description: 'Webhook for support emails'
})

// Email forward endpoint
const emailForward = await inbound.endpoints.create({
  name: 'Forward to Gmail',
  type: 'email',
  config: {
    forwardTo: 'team@gmail.com',
    preserveHeaders: true
  }
})

// Email group endpoint
const emailGroup = await inbound.endpoints.create({
  name: 'Team Distribution List',
  type: 'email_group',
  config: {
    emails: ['alice@team.com', 'bob@team.com', 'carol@team.com'],
    preserveHeaders: true
  }
})
```
</CodeGroup>

### POST `/api/e2/endpoints`

Create a new endpoint (webhook, email, or email_group) for the authenticated user.

#### Request Parameters

<ParamField path="name" type="string" required>
  Name of the endpoint (1-255 characters).
</ParamField>

<ParamField path="type" type="string" required>
  Endpoint type: `webhook`, `email`, or `email_group`.
</ParamField>

<ParamField path="config" type="object" required>
  Configuration object based on endpoint type.
  
  <Expandable title="Webhook Config">
    <ParamField path="url" type="string" required>
      Webhook URL to receive email notifications.
    </ParamField>
    
    <ParamField path="timeout" type="number" default="30">
      Request timeout in seconds (1-300).
    </ParamField>
    
    <ParamField path="retryAttempts" type="number" default="3">
      Number of retry attempts on failure (0-10).
    </ParamField>
    
    <ParamField path="headers" type="object">
      Custom headers to include with webhook requests.
      
      Example: `{ "X-Custom-Header": "value" }`
    </ParamField>
  </Expandable>
  
  <Expandable title="Email Config">
    <ParamField path="forwardTo" type="string" required>
      Email address to forward emails to.
    </ParamField>
    
    <ParamField path="preserveHeaders" type="boolean" default="false">
      Whether to preserve original email headers when forwarding.
    </ParamField>
  </Expandable>
  
  <Expandable title="Email Group Config">
    <ParamField path="emails" type="string[]" required>
      Array of email addresses (1-50 addresses).
    </ParamField>
    
    <ParamField path="preserveHeaders" type="boolean" default="false">
      Whether to preserve original email headers when forwarding.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="description" type="string">
  Optional description (max 1000 characters).
</ParamField>

#### Response

<ResponseField name="id" type="string">
  Unique identifier for the endpoint.
</ResponseField>

<ResponseField name="name" type="string">
  Endpoint name.
</ResponseField>

<ResponseField name="type" type="string">
  Endpoint type: `webhook`, `email`, or `email_group`.
</ResponseField>

<ResponseField name="config" type="object">
  Configuration object (includes `verificationToken` for webhooks).
</ResponseField>

<ResponseField name="isActive" type="boolean">
  Whether the endpoint is active.
</ResponseField>

<ResponseField name="description" type="string">
  Endpoint description.
</ResponseField>

<ResponseField name="userId" type="string">
  ID of the user who owns this endpoint.
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp when the endpoint was created.
</ResponseField>

<ResponseField name="updatedAt" type="string">
  ISO 8601 timestamp when the endpoint was last updated.
</ResponseField>

<ResponseField name="groupEmails" type="string[]">
  Array of group email addresses (only for email_group type).
</ResponseField>

<ResponseField name="deliveryStats" type="object">
  Delivery statistics.
  
  <Expandable title="Delivery Stats Object">
    <ResponseField name="total" type="number">
      Total number of delivery attempts.
    </ResponseField>
    
    <ResponseField name="successful" type="number">
      Number of successful deliveries.
    </ResponseField>
    
    <ResponseField name="failed" type="number">
      Number of failed deliveries.
    </ResponseField>
    
    <ResponseField name="lastDelivery" type="string">
      ISO 8601 timestamp of last delivery attempt.
    </ResponseField>
  </Expandable>
</ResponseField>

### GET `/api/e2/endpoints`

Get paginated list of endpoints for authenticated user with optional filtering by type, active status, sort order, and search by name.

#### Query Parameters

<ParamField query="limit" type="integer" default="50">
  Maximum number of endpoints to return (1-100).
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of endpoints to skip for pagination.
</ParamField>

<ParamField query="type" type="string">
  Filter by endpoint type: `webhook`, `email`, or `email_group`.
</ParamField>

<ParamField query="active" type="string">
  Filter by active status: `true` or `false`.
</ParamField>

<ParamField query="sortBy" type="string" default="newest">
  Sort order: `newest` or `oldest`.
</ParamField>

<ParamField query="search" type="string">
  Search by name or configuration content (max 100 characters).
</ParamField>

#### Response

<ResponseField name="data" type="array">
  Array of endpoint objects (same structure as Create Endpoint response).
</ResponseField>

<ResponseField name="pagination" type="object">
  Pagination metadata.
  
  <Expandable title="Pagination Object">
    <ResponseField name="limit" type="number">
      Number of results per page.
    </ResponseField>
    
    <ResponseField name="offset" type="number">
      Number of results skipped.
    </ResponseField>
    
    <ResponseField name="total" type="number">
      Total number of matching endpoints.
    </ResponseField>
    
    <ResponseField name="hasMore" type="boolean">
      Whether there are more results available.
    </ResponseField>
  </Expandable>
</ResponseField>

## Update Endpoint

<CodeGroup>
```bash cURL
curl -X PATCH https://api.inbound.new/api/e2/endpoints/ep_abc123 \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Updated Support Webhook",
    "config": {
      "url": "https://api.example.com/webhooks/new-email",
      "timeout": 60
    },
    "isActive": true
  }'
```

```typescript TypeScript SDK
const endpoint = await inbound.endpoints.update('ep_abc123', {
  name: 'Updated Support Webhook',
  config: {
    url: 'https://api.example.com/webhooks/new-email',
    timeout: 60
  },
  isActive: true
})
```
</CodeGroup>

## Webhook Verification

All webhook endpoints automatically receive a `verificationToken` in their config. Use this token to verify that webhook requests are coming from Inbound:

```typescript
// Verify webhook signature
const signature = request.headers.get('X-Inbound-Signature')
const verificationToken = 'wh_verify_abc123def456' // From endpoint config

if (signature !== verificationToken) {
  return new Response('Invalid signature', { status: 401 })
}

// Process the email...
```
